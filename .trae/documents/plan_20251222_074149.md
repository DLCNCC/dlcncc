# 通知插件配置方案

## 1. 需求分析

根据用户提供的主题通知插件文档，需要在VuePress主题中添加通知功能，支持为不同路径设置多个公告。

## 2. 通知插件配置结构

根据文档，通知插件配置需要添加在主题配置的`plugins.notice`属性中，支持以下配置项：

| 配置项 | 类型 | 说明 |
|--------|------|------|
| path/match | string/regex | 匹配路径，path为字符串匹配开头，match为正则表达式匹配 |
| title | string | 通知标题，支持文本和HTML |
| content | string | 通知内容，支持文本和HTML |
| actions | array | 通知操作按钮配置数组 |
| showOnce | boolean | 仅显示一次，关闭后不再显示 |
| key | string | 通知记忆键值，用于编辑通知而不打扰已确认用户 |
| fullscreen | boolean | 全屏弹出窗口 |
| confirm | boolean | 仅允许通过操作按钮关闭通知 |

## 3. 实施步骤

### 3.1 添加通知插件配置

在主题配置的`plugins`对象中添加`notice`配置，示例如下：

```typescript
export default hopeTheme({
  // 其他配置...
  
  plugins: {
    // 其他插件配置...
    
    // 通知插件配置
    notice: [
      {
        path: "/",
        title: "站点通知",
        content: "欢迎访问我们的文档站点！",
        actions: [
          {
            text: "了解更多",
            link: "https://dlcn.cc",
            type: "primary",
          },
          { text: "关闭" },
        ],
        showOnce: true,
      },
    ],
  },
});
```

### 3.2 详细配置说明

1. **基础通知配置**：为根路径添加欢迎通知
2. **多路径支持**：可以为不同路径添加不同通知
3. **通知操作**：添加主操作和关闭操作
4. **显示控制**：设置`showOnce: true`，用户关闭后不再显示
5. **可选配置**：根据需要添加`fullscreen`、`confirm`等高级选项

```typescript
          {

      },
      {

          {

            type: "primary",

          { text: "关闭" },
        ],
      },

    // 其他插件配置...
  },

```

## 5. 预期效果

- 访问站点根路径时，会显示欢迎通知
- 用户关闭通知后，再次访问将不再显示（由于设置了`showOnce: true`）
- 通知包含标题、内容和两个操作按钮
- 支持多路径不同通知配置
